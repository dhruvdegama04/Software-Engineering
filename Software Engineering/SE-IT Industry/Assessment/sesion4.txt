1. List all 6 phases of SDLC in order with one line about each.

Phase				Description
1. Requirement Analysis		Gather and understand the customer's requirements.
2. System Design		Create the architecture and design of the software.
3. Development (Coding)		Write the program code based on the design.
4. Testing			Check the software for bugs and ensure it works correctly.
5. Deployment			Release the software to users.
6. Maintenance			Fix bugs, update features, and improve the software after release.

2. Write 3 differences between Agile and Waterfall models.

Agile								Waterfall
Development is done in small iterations (sprints).		Development follows one fixed sequence of phases.
Changes can be made easily during the project.			Changes are difficult after a phase is completed.
Customer feedback is received continuously.			Customer feedback is usually taken at the end of 								the project.

3. Name the 3 main Scrum roles and write one line about the responsibility of each.

Scrum Role		Responsibility
Product Owner		Defines product requirements and prioritizes the work.
Scrum Master		Guides the Scrum process and removes obstacles for the team.
Development Team	Designs, develops, and tests the product.

4. What is a Sprint in Scrum? Mention its typical duration.

A Sprint is a fixed period during which the Scrum team completes a set of planned tasks.

Typical Duration: 1 to 4 weeks (most commonly 2 weeks).

5. What do these Jira board columns mean?
Column		Meaning
Backlog		List of tasks that are planned but not yet started.
To Do		Tasks that are ready to begin.
In Progress	Tasks that are currently being worked on.
Done		Tasks that have been completed successfully.
